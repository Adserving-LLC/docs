# RDF: Стандартный файл - Standard File

[< Файлы сырых данных (RDF - Raw Data File)](../RDF.md)

Описание полей:
В следующей таблице описаны стандартные поля необработанных данных (RDF).

|Название|Описание|Тип|
|-|-|-|
|CampaignID|Уникальный идентификатор кампании Adserving.|Integer|
|EntityID|Уникальный идентификатор объявления Adserving.|Integer|
|EventDate|Дата, когда событие было вызвано.|DATETIME|
|EventTypeID|Уникальный идентификатор типа события, который будет отображаться в стандартном канале и соответствует файлу метаданных. Доступны следующие варианты, соответствующие агрегированные показатели отчетности указаны в скобках: </br>1: Impression (показы очищенные от GIVT) </br>2: Click (клики очищенные от GIVT)|Integer|
|PlacementID|Уникальный идентификатор рекламного объявления.|Integer|
|SiteID|Уникальный идентификатор сайта, на котором размещается рекламное объявление.|Integer|
|AccountID|Уникальный идентификатор аккаунта Adserving.|Integer|
|AdvertiserDeviceID|Уникальный идентификатор мобильного устройства (IDFA, GAID, WAID). Рекламный сервис получает это значение только в том случае, если паблишер отправляет его в мобильном токене.|Integer|
|AdvertiserDeviceType|Возможные значения: IDFA GAID WAID empty Adserving получает это значение, только если паблишер отправляет его в мобильном токене.|String|
|AdvertiserID|Уникальный идентификатор рекламодателя Adserving.|Integer|
|AreaCode|Код телефонного города, в котором произошло событие.|VARCHAR(32)|
|AudienceID|Уникальный идентификатор, присвоенный Adserving при создании целевой аудитории.|Integer|
|BrandID|Уникальный идентификатор бренда Adserving.|Integer|
|BrowserCode|Уникальный идентификатор типа браузера (как он указан в файле метаданных), в котором произошло событие.|Integer|
|CityID|ID города, в котором произошло событие. Название города можно найти в файле метаданных.|Integer|
|CountryID|Уникальный идентификатор страны, в которой было инициировано событие. Эти идентификаторы отображаются в файле метаданных.|Integer|
|DeliveryGroupId|Уникальный идентификатор группы рекламных объявлений Adserving.|Integer|
|DeviceType|Типы устройств. Возможные значения: Smartphone, Mac, PC, SmartTV, Tablet, Streaming TV, Other|String|
|DMAID|ID DMA (только для DMA в США), где произошло событие. Имя DMA можно найти в файле метаданных.|Integer|
|Environment|Среда, в которой произошло событие (в приложении, в web версии, не задано).|String|
|ImpressionTypeID|0: было показано обычное объявление.</br> 1: было предоставлено изображение по умолчанию.|Integer|
|IP|IP пользовательского устройства, вызвавшего событие; актуально для всех потоков данных с IP. </br> Примечание: В соответствии с регламентом GDPR последний октет IP-адреса усекается. Это поле пусто для немецких IP-адресов.|String (16)|
|MobileDevice|Название мобильного устройства, на котором было показано объявление. Это значение представляет собой объединение названия бренда и названия модели, например Apple iPad 9.7, Samsung sm-g965f.|String (100)|
|OSCode|Уникальный идентификатор типа операционной системы, который можно найти в файле метаданных. Указывает тип операционной системы на устройстве, вызвавшем событие.|Integer|
|PCP|Пользовательский параметр паблишера (PCP). Параметр, определенный для издателя/сайта, отображается только в том случае, если PCP существует.|VARCHAR (2048)|
|Referrer|URL-адрес, полученный из поля реферера HTTP-заголовка в процессе показа/отслеживания рекламы. Может быть URL-адресом страницы, на которой появилось объявление, URL-адресом рекламного сервера или пустым, если оно не было передано.|String|
|SECampaignID|Уникальный идентификатор кампании на платформе поисковой системы.|Integer|
|SEMatchType|Отображает релевантность поискового запроса: Broad (Широкое), Broad match modifier (Модификатор широкого соответствия), Phrase (Фразовое), Exact (Точное)|String|
|SEPosition|Числовое значение, описывающее положение результата поиска на странице.|Integer|
|SessionID|Уникальный идентификатор сеанса. Позволяет связать показ, клик и взаимодействие, принадлежащие одному сеансу.|Integer|
|StateID|Уникальный идентификатор состояния, в котором произошло событие. Название состояния можно найти в файле метаданных.|Integer|
|UserID|Уникальный идентификатор пользователя Adserving (Stable ID).|String (36)|
|VersionID|Уникальный идентификатор версии DCO кратива Adserving|Integer|
|ZipCode|Почтовый индекс, в котором произошло событие.|String (32)|
|UserConsent|Закодированная строка с сайта паблишера, определяющая поставщиков и цели, на которые пользователь дает согласие.|String|
