# RDF: Конверсионный файл - Conversion File

[< Файлы сырых данных (RDF - Raw Data File)](../RDF.md)

Описание полей:
В следующей таблице описаны поля фида конверсий (RDF). Фид конверсий включает только данные о выигрышных событиях из основной модели конверсий соответствующих рекламодателей.

|Название|Описание|Тип|
|-|-|-|
|AccountID|Уникальный идентификатор аккаунта Adserving.|Integer|
|AdvertiserDeviceID|Уникальный идентификатор мобильного устройства (IDFA, GAID, WAID). Adserving получает это значение только в том случае, если паблишер отправляет его в мобильном токене.|String|
|AdvertiserDeviceType|Уникальный идентификатор мобильного устройства (IDFA, GAID, WAID). Adserving получает это значение только в том случае, если паблишер отправляет его в мобильном токене.|String|
|AdvertiserID|Уникальный идентификатор рекламодателя Adserving.|Integer|
|AreaCode|Телефонный код города.|VARCHAR(32)|
|BrandID|Уникальный идентификатор бренда Adserving,связанный с объявлением, которое привело к конверсии|Integer|
|BrowserCode|Коды браузера, которые можно найти в файле метаданных.|Integer|
|CampaignID|Уникальный идентификатор кампании Adserving, связанный с объяклением, которое привело к конверсии.|Integer|
|CityID|-|-|
|ConversionActivityID|Уникальный идентификатор, присвоенный действию конверсии при его создании Adserving.|Integer|
|ConversionDate|Дата, когда произошла конверсия. Формат: Year-Month-Day / Hour:Minute:Second|DateTime|
|ConversionID|Уникальный идентификатор, присваиваемый действию-конверсии при записи.|String (36)|
|CountryID|ID страны, в которой произошло событие. Эти идентификаторы отображаются в файле страны метаданных.|Integer|
|Currency|Цифровой код валюты, в которой произошла конвертация. Название валюты можно найти в файле метаданных валюты.Валюта будет соответствовать валюте, установленной в соответствующем конверсионном действии.|Integer|
|CustomParam 1..49 |Значение настраиваемого параметра, полученное из тега преобразования, как определено Adserving. Система поддерживает до 49 пользовательских параметров для каждого действия конверсии.|String (200)|
|DeviceTypeID|Возможные значения:</br> 1: PC</br> 2: Mac </br>3: Tablet </br>4: Smart Phone </br>5: Other </br>6: Smart TV </br>7: OTT|Integer|
|DMAID|Уникальный идентификатор DMA (только для DMA в США), где произошло событие. Имя DMA можно найти в файле метаданных.|Integer|
|EventTypeID|Уникальный идентификатор типа победившего события, который можно найти в файле метаданных.|Integer|
|Environment|Среда, в которой произошло выигрышное событие (в приложении, в web версии, не задано).|String|
|IP|IP пользовательского устройства, вызвавшего событие. Примечание. В соответствии с регламентом GDPR последний октет IP-адреса усекается. Это поле пусто для немецких IP-адресов.|String (16)|
|IsConversion|True, если действие преобразования имеет какое-либо событие по заданому пути к преобразованию. False, если по заданому пути нет событий.|Boolean|
|OrderID|Уникальный идентификатор, присвоенный рекламодателем заказу, записанный тегом конверсии продаж.|String|
|OSCode|Уникальный идентификатор типа операционной системы, который можно найти в файле метаданных.|Integer|
|PCP|Пользовательский параметр паблишера (PCP). Параметр, определенный для паблишера/сайта, отображается только в том случае, если PCP существует.|String (2048)|
|PlacementID|Уникальный идентификатор места размещения выигравшего события.|Integer|
|ProductId|Уникальный идентификатор, назначенный рекламодателем продукту, записанный тегом конверсии продаж.|String|
|ProductInfo|Информация о продукте, определенная рекламодателем и зафиксированная тегом конверсии продаж.|String (135)|
|Quantity|Общее количество заказанного продукта, зафиксированное в ходе конверсии продаж.|Integer|
|QueryString|Вся информация, которую тег активности Adserving записал и передал Adserving при совершении преобразования.|String|
|Referrer|URL-адрес предыдущей страницы, которую посетил пользователь.|String|
|Revenue|Значение, зафиксированное тегом конверсии продаж.|Float|
|SEMatchType|Отображает релевантность поискового запроса:  Broad (Широкое), Broad match modifier (Модификатор широкого соответствия), Phrase (Фразовое), Exact (Точное)|String|
|SEPosition|Числовое значение, описывающее положение результата поиска на странице.|Integer|
|SiteID|Уникальный идентификатор сайта, присвоенный месту размещения победившего объявления.|Integer|
|StateID|ID состояния, в котором произошло событие. Название состояния можно найти в файле метаданных.|Integer|
|UserID|Уникальный идентификатор файла cookie Adserving.|String (36)|
|WinnerDeliveryGroupID|Уникальный идентификатор группы креативов, к которой прикреплено выигравшее объявление.|Integer|
|WinnerEntityID|Уникальный идентификатор объявления для победившего события.|Integer|
|WinnerEventDate|Отметка времени победившего события.|Date/Time|
|WinnerSECampaignID|Уникальный идентификатор кампании в поисковой платформе.|Integer|
|WinnerSessionID|Уникальный идентификатор сессии события-победителя.|String (36)|
|WinnerSearchAdGroupID|Уникальный идентификатор группы объявлений Adserving.|Integer|
|WinnerSearchCampaignID|Уникальный идентификатор кампании Adserving.|Integer|
|WinnerTargetAudienceID|Уникальный идентификатор целевой аудитории, используемый при таргетинге на событие-победитель.|Integer|
|WinnerVersionID|Применяется к динамическим креативным объявлениям. Уникальный идентификационный номер, присвоенный Adserving при создании версии.|Integer|
|ZipCode|Почтовый индекс, в котором произошло событие.|String (32)|
|WinnerDeliveryGroupIDcld|Зарезервирован. В настоящий момент всегда установлен как -1.|String (32)|
|WinnerTargetAudienceIDcld|Зарезервирован. В настоящий момент всегда установлен как -1.|String (32)|
|UserConsent|Закодированная строка с сайта паблишера или рекламодателя, определяющая поставщиков и цели, на которые пользователь дает согласие. В файле канала это будет отображаться в виде двух полей: <br>UserConsentConversion: строка согласия пользователя на действие. <br>UserConsentWinner: согласие пользователя передается в победном событии. <br>Примечание: Если паблишер внедрил структуру согласия и передает нам согласие пользователя, а рекламодатель не внедрил структуру согласия, не собрал и не передал нам согласие пользователя, UserConsentWinner будет содержать информацию о согласии, а UserConsentConversion будет иметь пустое поле.|String|
